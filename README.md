# 통합 인쇄 제작 OS (타디스)

스티커 / 엽서 / 마스킹테이프 인쇄 업체의 단가를 등록해두면, 주문 조건(사이즈·수량·옵션)에 맞춰
등록된 업체 중 최저가를 자동으로 비교해주는 Streamlit 앱.

## 1. 로컬에서 실행하기

```bash
pip install -r requirements.txt
python -m streamlit run app.py
```

또는 `인쇄프로그램실행.bat` 더블클릭.

구글 시트 연동을 설정하지 않으면 업체 데이터는 이 폴더의 `vendor_data.json` 파일에만 저장된다.
(같은 PC에서 혼자 쓸 때는 이 상태로 충분하다.)

## 2. 여러 사람이 같은 업체 데이터를 공유하려면 (구글 시트 연동)

설정 콘솔에서 업체를 등록/수정하면, 이 앱을 쓰는 **모든 사용자**에게 그 데이터가 반영되도록
하려면 구글 시트를 공유 저장소로 연결해야 한다. 한 번만 설정하면 된다.

### 2-1. 구글 클라우드에서 서비스 계정 만들기

1. [Google Cloud Console](https://console.cloud.google.com/)에서 새 프로젝트를 만든다.
2. **API 및 서비스 > 라이브러리**에서 `Google Sheets API`와 `Google Drive API`를 각각 검색해 사용 설정한다.
3. **API 및 서비스 > 사용자 인증 정보 > 사용자 인증 정보 만들기 > 서비스 계정**으로 서비스 계정을 만든다.
4. 만든 서비스 계정에서 **키 추가 > 새 키 만들기 > JSON**을 선택해 키 파일을 다운로드한다.
5. 다운로드한 JSON 안의 `client_email` 값(예: `xxx@xxx.iam.gserviceaccount.com`)을 기억해둔다.

### 2-2. 업체 데이터를 저장할 구글 시트 준비

1. 구글 드라이브에서 새 스프레드시트를 만든다. (시트 안의 내용은 앱이 자동으로 채우므로 비워둬도 된다.)
2. 시트 우측 상단 **공유** 버튼을 눌러, 위에서 확인한 `client_email`을 **편집자**로 초대한다.
3. 시트 주소창의 전체 URL을 복사해둔다.

### 2-3. 앱에 인증 정보 연결하기

`.streamlit/secrets.toml.example` 파일을 복사해 `.streamlit/secrets.toml`로 저장하고,
`SHEET_URL`에 2-2에서 복사한 시트 주소를, `[gcp_service_account]` 아래에는 2-1에서 받은
JSON 키 파일의 값을 그대로 옮겨 적는다. (`secrets.toml`은 `.gitignore`에 포함되어 있어
깃 저장소에는 올라가지 않는다 — 비밀키이므로 절대 커밋하지 않는다.)

이후 앱을 실행하면 설정 콘솔 상단에 "🔗 구글 시트 연동 중" 문구가 뜨면 정상 연결된 것이다.
연동 후에는 업체 등록/수정/삭제가 모두 이 구글 시트에 저장되며, 다른 사용자는 설정 콘솔의
"🔄 구글 시트에서 최신 업체 데이터 새로고침" 버튼으로 반영받을 수 있다.

## 3. Streamlit Community Cloud로 배포하기

1. 이 폴더를 GitHub 저장소로 올린다. (`secrets.toml`은 올리지 않는다.)
2. [share.streamlit.io](https://share.streamlit.io/)에서 New app > 방금 올린 저장소를 선택하고,
   Main file path는 `app.py`로 지정한다.
3. 앱 설정의 **Secrets** 탭에 `.streamlit/secrets.toml.example`과 같은 형식으로
   실제 `SHEET_URL`과 `gcp_service_account` 값을 붙여넣는다.
4. 배포하면 발급되는 URL을 여러 사람에게 공유하면 된다.

### 알아둘 점

- **업체 데이터**(스티커/엽서/마스킹테이프 단가표)는 구글 시트를 연동하면 모든 사용자가 공유한다.
- 반면 **화면 설정**(`config.json`), **마스터 옵션 목록**(`master_options.json`), **오류 로그**(`error_log.txt`)는
  아직 서버의 로컬 파일에만 저장된다. Streamlit Community Cloud는 코드가 바뀌거나 앱이 재시작되면
  로컬 파일이 초기화되므로, 이 세 가지는 재배포 시 기본값으로 되돌아갈 수 있다.
  (업체 데이터처럼 공유/영구 저장이 필요해지면 같은 방식으로 구글 시트를 추가 연동하면 된다.)

## 4. 상품군과 입력 양식

업체 등록 콘솔의 **➕ 새 상품군 추가**에서 상품군을 만들 때 입력 양식을 고른다.

| 양식 | 언제 쓰나 | 입력 항목 |
|---|---|---|
| 사이즈별 단가표 | 아크릴 굿즈처럼 **종류·사이즈마다 단가가 다른** 경우 | 세부 종류, 사이즈 목록, 수량 구간, 종류별 사이즈 × 수량 단가표 |
| 스티커형 | 용지/접착/후지/코팅 조합으로 단가가 정해지는 경우 | 재료 옵션, 조합별 구간 단가 |

기본 탑재된 `스티커` / `엽서` / `마스킹 테이프`는 각각 전용 양식이 고정되어 있다.

### 아크릴 굿즈 등록 방법

아크릴은 굿즈 종류(키링·코롯토·스마트톡…)마다 취급 사이즈와 단가표가 완전히 다르므로,
**굿즈 종류별로 상품군을 따로 만든다.** 예를 들어 `아크릴키링`, `코롯토`를 각각 추가하고
양식은 **사이즈별 단가표**를 고른다.

그 다음 업체를 등록할 때 업체 페이지의 단가표를 그대로 옮긴다:

1. **세부 종류** — 예: `[일반] 투명`, `[라미] 투명`, `하프미러`, `글리터`, `자개`
2. **사이즈 목록** — 예: `20x20`, `30x15`, `30x30`, `40x40` …
3. **수량 구간** — 예: `1-9, 10-99, 100-499, 500-999`
4. **단가표** — 종류별로 `수량 구간 × 사이즈` 표가 생기니 개당 단가를 채운다.
   업체 사이트 표의 숫자 영역을 복사해 붙여넣을 수도 있다.
   **0으로 두면 그 사이즈는 취급하지 않는 것으로 처리**되어 검색 결과에서 빠진다.

## 5. 파일 구성

| 파일 | 역할 |
|---|---|
| `app.py` | 앱 전체 (설정 콘솔 + 주문 비교 화면) |
| `vendor_data.json` | 업체 데이터 (구글 시트 미연동 시 로컬 저장소) |
| `config.json` | 화면 색상/배치 설정 |
| `master_options.json` | 용지·접착·코팅 등 마스터 옵션 목록 (실행 중 자동 생성) |
| `.streamlit/secrets.toml.example` | 구글 시트 연동용 비밀키 입력 양식 |
